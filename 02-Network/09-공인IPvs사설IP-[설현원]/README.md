# 네트워크 기본 개념 정리

본 문서는 **공인 IP와 사설 IP**, **서브네팅(Subnetting)**, **CIDR 표기법**, 그리고 **IP 주소 클래스**에 대한 발표 자료 내용을 요약한 것입니다.

---

## 1. IP란?
- **IP(Internet Protocol Address)**: 인터넷에서 기기(컴퓨터, 스마트폰, 서버 등)를 구분하기 위한 고유 주소
- **역할**: 전화번호나 집 주소처럼 ‘누가 누구인지’ 식별
- **형식**
  - IPv4 예시: `192.168.0.1`
  - IPv6 예시: `2001:db8::1`

---

## 2. 공인 IP (Public IP)
- 인터넷 상에서 **전 세계적으로 유일하게 식별 가능**한 주소
- **ISP(Internet Service Provider)**가 제공
- 진정한 의미의 인터넷 주소

### 특징
- 고갈 문제 (IPv4 부족)
- 보안 문제 (공개되어 공격 노출 가능)
- 관리 효율성 저하
- NAT 필요성 증가

---

## 3. 사설 IP (Private IP)
- 가정/회사 **내부 네트워크에서만 사용되는 주소**
- 외부와 통신할 때는 공유기(NAT)가 공인 IP로 변환 후 전달

### 장점
- 공인 IP 부족 문제 해결
- 내부 기기 보호 (보안 강화)
- 네트워크 관리 간편화

---

## 4. NAT (Network Address Translation) 동작
- **내부 → 외부 요청**
  - `192.168.0.2:5001 → 8.8.8.8:80`
  - 공유기가 변환: `125.176.20.5:40001 → 8.8.8.8:80`
- **외부 → 내부 응답**
  - `8.8.8.8:80 → 125.176.20.5:40001`
  - 공유기가 NAT 테이블 참조 후 `192.168.0.2:5001`로 변환

---

## 5. 서브네팅 (Subnetting)
- 큰 네트워크를 여러 개의 작은 네트워크로 나누는 기술
- **주소 낭비를 줄이고 관리 효율성을 높임**

### 예시
- `/24` → 앞의 24비트는 네트워크, 뒤 8비트는 호스트

---

## 6. CIDR 표기법
- **CIDR(Classless Inter-Domain Routing)**
- IP 주소 + 프리픽스 길이로 네트워크를 표현
  - 예: `192.168.1.0/24`

### 서브넷 마스크 예시
- `/24` → `255.255.255.0`
- 이진수: `11111111.11111111.11111111.00000000`

---

## 7. 네트워크 예약 주소
- **네트워크 주소**: 호스트 비트가 전부 0
  - 예: `192.168.1.0/24`
- **브로드캐스트 주소**: 호스트 비트가 전부 1
  - 예: `192.168.1.255/24`

---

| CIDR | 서브넷 마스크      | 사용 가능한 호스트 수 |

| /25  | 255.255.255.128   | 126                 |

---

## 9. IP 주소 클래스 (역사적 배경)

| 클래스 | 시작 비트 | 주소 범위              | 기본 서브넷 마스크 | 호스트 수 |
|--------|-----------|-----------------------|-------------------|-----------|
| A      | 0xxx      | 0.0.0.0 ~ 127.255.255.255 | 255.0.0.0 (/8)   | 약 1,600만 |
| B      | 10xx      | 128.0.0.0 ~ 191.255.255.255 | 255.255.0.0 (/16) | 약 65,000 |
| C      | 110x      | 192.0.0.0 ~ 223.255.255.255 | 255.255.255.0 (/24) | 254 |
| D      | 1110      | 224.0.0.0 ~ 239.255.255.255 | 멀티캐스트용     | - |
| E      | 1111      | 240.0.0.0 ~ 255.255.255.255 | 연구/예약용      | - |

### 현재는?
- CIDR 도입으로 네트워크 부분 길이를 유동적으로 조정 가능
- 클래스 개념은 **역사적 배경**으로만 존재

---

## 📌 결론
- **공인 IP**: 인터넷 상 유일, ISP가 부여, 보안/고갈 문제 존재  
- **사설 IP**: 내부 네트워크용, NAT로 외부 연결, 보안·관리 효율성↑  
- **서브네팅/CIDR**: IP 주소 효율적 사용, 관리성 강화  
- **클래스 방식**: 현재는 역사적 유산, 대신 CIDR 활용
