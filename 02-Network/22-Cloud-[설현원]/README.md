# ☁️ 클라우드 컴퓨팅(Cloud Computing)

## 1. 개요

**클라우드 컴퓨팅(Cloud Computing)** 은  
인터넷을 통해 서버, 저장소, 네트워크, 애플리케이션 등의 IT 자원을  
필요한 만큼 빌려 쓰는 컴퓨팅 서비스 모델입니다.

> “내 컴퓨터가 아니라, 인터넷 어딘가의 거대한 컴퓨터를 빌려 쓰는 것.”

---

## 2. 클라우드 컴퓨팅의 핵심 원리

| 개념 | 설명 |
|------|------|
| **가상화 (Virtualization)** | 물리 자원을 여러 가상 자원으로 분할해 효율적으로 활용. 예: 하나의 서버를 여러 VM으로 운영 |
| **리소스 풀링 (Resource Pooling)** | 여러 사용자가 하나의 자원 풀을 공유하지만, 논리적으로는 분리되어 보안 유지 |
| **자동화 및 오케스트레이션 (Automation & Orchestration)** | 자원의 생성·배포·확장을 자동으로 수행 (예: Kubernetes) |
| **유연한 확장성 (Elasticity)** | 사용량에 따라 자원을 자동으로 늘리거나 줄임 (Auto Scaling) |
| **측정 가능한 서비스 (Measured Service)** | 사용량을 자동 측정하고, 사용량만큼 과금 (Pay-as-you-go) |

---

## 3. 가상화 (Virtualization)

### 핵심: 하이퍼바이저(Hypervisor)

하이퍼바이저는 물리 하드웨어 위에서 가상 머신(VM)을 생성·관리하는 소프트웨어입니다.

[사용자 OS1] [사용자 OS2] [사용자 OS3]
↓ ↓ ↓
┌───────────────────────────┐
│ 하이퍼바이저 │ ← 가상화 제어 계층
└───────────────────────────┘
↓ ↓ ↓
┌────────────────────────┐
│ 물리 서버 (CPU, RAM, Disk, NIC) │
└────────────────────────┘


- 각 VM은 **독립적인 OS**와 환경을 가짐  
- 사용자는 자신만의 서버를 쓰는 것처럼 인식  
- **Type 1 (베어메탈)**: VMware ESXi, KVM, Hyper-V  
- **Type 2 (호스트형)**: VirtualBox

---

## 4. 리소스 풀링 (Resource Pooling)

> “많은 사용자가 하나의 자원 풀을 함께 쓰지만,  
> 각자 자신만의 서버를 사용하는 것처럼 느낀다.”

| 구성 요소 | 역할 |
|------------|------|
| **오케스트레이터** | 자원 배분 및 회수 자동화 (예: Kubernetes, OpenStack) |
| **로드 밸런서** | 트래픽을 여러 서버에 분산 |
| **가상 네트워킹(SDN)** | 사용자의 네트워크 환경을 논리적으로 격리 |
| **리소스 스케줄러** | 우선순위, 사용량에 따른 자원 자동 할당 |

멀티테넌시(Multi-tenancy)를 통해  
여러 사용자가 인프라를 공유하되 **보안과 안정성**을 확보합니다.

---

## 5. 자동화와 오케스트레이션

| 개념 | 설명 |
|------|------|
| **자동화 (Automation)** | 특정 작업을 수동 개입 없이 자동으로 처리 (VM 생성, 배포 등) |
| **오케스트레이션 (Orchestration)** | 여러 자동화된 작업을 조율해 하나의 완성된 프로세스로 만듦 |

예시:  
“웹 서버 3대를 늘려줘” 요청 시  
1️⃣ 오케스트레이터가 인식  
2️⃣ 자동화 엔진이 VM 생성  
3️⃣ 네트워크/로드밸런서 설정  
4️⃣ 모니터링 시스템 등록  
→ 전체가 몇 초 내 자동 완료

## 6. 유연한 확장성 (Elasticity)

> 필요할 때 늘리고, 필요 없을 때 줄인다.

| 구분 | 온프레미스 | 클라우드 |
|------|-------------|-----------|
| 자원 확보 | 미리 서버 구매 (고정) | 즉시 확장 (유동) |
| 비용 | 항상 최대치 기준 | 사용량 기반 과금 |
| 속도 | 수동 설정 | 자동 스케일링 |
| 확장 방식 | 물리 장비 추가 | 클릭/정책 기반 자동화 |

**수평 확장 (Scale-out)** 방식이 일반적  
→ 자동화와 부하 분산이 쉬움, 장애 복구 용이

## 7. 측정 가능한 서비스 (Measured Service)

클라우드는 사용량을 실시간 측정하고 이에 따라 과금합니다.

| 서비스 | 과금 기준 | 예시 |
|----------|-------------|------|
| **AWS EC2** | 인스턴스 사용 시간 | 초 단위 과금 |
| **GCP Storage** | 저장 데이터 용량 | GB 단위 과금 |
| **Azure Functions** | 함수 실행 횟수 | 100만 회 기준 |
| **OpenAI API** | 토큰 수 | 입력·출력 토큰 단위 |
| **Netflix CloudWatch** | 로그 수집량 | GB 단위 요금 |

## 8. 클라우드 계층 구조

| 계층 | 구성 요소 | 예시 |
|------|------------|------|
| **물리 인프라** | 서버, 스토리지, 네트워크, 전력 | Dell 서버, Cisco 스위치 |
| **가상화 계층** | 하이퍼바이저, VM | VMware, KVM |
| **운영체제(OS)** | OS 환경 | Linux, Windows Server |
| **미들웨어** | DBMS, WAS, MQ | MySQL, Tomcat, Kafka |
| **런타임** | 언어·프레임워크 실행 환경 | JVM, Node.js, Docker |
| **데이터 계층** | DB·스토리지 | S3, DynamoDB, Redis |
| **애플리케이션** | 사용자 앱 | Gmail, Zoom, Slack |

## 9. 클라우드 서비스 모델 비교

| 구분 | **IaaS** | **PaaS** | **SaaS** |
|------|-----------|-----------|-----------|
| 개념 | 인프라를 빌려씀 | 개발 플랫폼을 빌려씀 | 소프트웨어를 빌려씀 |
| 관리 주체 | 사용자가 직접 관리 | 제공자 일부 관리 | 제공자 전담 관리 |
| 사용 예 | AWS EC2, Google Compute | AWS Elastic Beanstalk, Heroku | Gmail, Slack, Zoom |
| 사용자 대상 | 시스템 관리자 | 개발자 | 일반 사용자 |
| 장점 | 자유도 높음, 유연한 설정 | 개발 생산성 향상 | 유지보수 無, 즉시 사용 |
| 단점 | 설정 복잡 | 플랫폼 종속 가능성 | 커스터마이징 제한 |
